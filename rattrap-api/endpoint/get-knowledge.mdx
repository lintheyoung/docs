## 使用说明

通过此接口可以获取单个知识库条目的详细信息（元数据）。支持获取文本、图片等各种类型的知识内容。

<Info>
**获取完整文件内容**：
- 此接口返回知识条目的元数据和文本描述
- 若要下载完整原始文件，请使用 `GET /knowledge-files/{id}` 接口
- `file_url` 字段提供了完整文件的访问链接
</Info>

<CardGroup cols={3}>
  <Card title="元数据查询" icon="file-lines">
    返回知识条目的所有元数据和内容描述
  </Card>
  <Card title="权限检查" icon="shield">
    自动验证用户对知识条目的访问权限
  </Card>
  <Card title="使用统计" icon="chart-bar">
    返回知识条目的使用次数等统计信息
  </Card>
</CardGroup>

<Warning>
**权限要求**：需要登录认证。只能访问有权限的知识条目（基于 `visibility` 和 `access_level`）。
</Warning>

## 请求参数

### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 知识库条目 ID，格式：`kb_xxx` |

## 访问权限规则

系统会根据 `visibility` 和 `access_level` 字段进行双层权限验证：

### 第一层：visibility 权限

<Info>
**visibility 检查逻辑**：
1. `visibility='public'` 的知识：所有已登录用户均可访问元数据
2. `visibility='private'` 的知识：仅同租户用户可访问元数据
3. 管理员可以访问所有知识（包括其他租户的私有知识）
</Info>

### 第二层：access_level 权限

当用户尝试访问 `file_url`（完整文件）时，根据 `access_level` 进行权限验证：

| access_level | 权限要求 | 验证逻辑 |
|--------------|----------|----------|
| `public` | 无需额外验证 | 任何已登录用户可访问 |
| `internal` | 需要登录 | 验证用户已登录，拒绝匿名访问 |
| `private` | 需要管理员角色 | 验证用户为 admin 或 owner 角色 |

**权限验证流程示例**：

```
1. 用户请求 GET /knowledge-base/kb_xxx
   ↓ 检查 visibility
   ✅ public 或同租户 → 返回元数据（包含 file_url）

2. 用户点击 file_url 链接
   ↓ 检查 access_level
   - public → ✅ 直接返回文件
   - internal → 检查登录 → ✅ 返回文件 / ❌ 401
   - private → 检查管理员 → ✅ 返回文件 / ❌ 403
```

<Warning>
**安全提示**：`file_url` 链接包含权限验证。即使用户获得了链接，也必须满足 `access_level` 要求才能访问文件内容。
</Warning>

## 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 知识库条目 ID，格式：`kb_xxx` |
| `tenant_id` | string | 租户 ID（private 模式）或 `null`（public 模式） |
| `title` | string | 知识标题 |
| `content` | string | 知识内容（文本或图片描述） |
| `content_type` | string | 内容类型：`document` / `image` / `text` / `qa` |
| `source_file_name` | string | 原始文件名 |
| `source_file_type` | string | 文件类型：`txt` / `jpg` / `png` / `pdf` / `docx` |
| `image_url` | string | 图片公开 URL（仅图片类型） |
| `file_url` | string | 文件访问 URL（用于获取完整原始文件） |
| `source_url` | string | 原始来源 URL（用户提供的外部链接） |
| `source_title` | string | 来源标题 |
| `access_level` | string | 访问权限级别：`public` / `internal` / `private` |
| `category` | string | 知识分类 |
| `tags` | string[] | 标签数组 |
| `language` | string | 语言代码 |
| `visibility` | string | 隐私级别：`private` / `public` |
| `is_active` | boolean | 是否启用 |
| `usage_count` | integer | 使用次数 |
| `created_by` | string | 创建者用户 ID |
| `created_at` | string | 创建时间（ISO 8601） |
| `updated_at` | string | 更新时间（ISO 8601） |

<Info>
**file_url 说明**：
- 对于切片文件，所有切片返回相同的 `file_url`（指向完整文件）
- 访问 `file_url` 会触发 `access_level` 权限验证
- 使用 `GET /knowledge-files/{id}` 可以下载完整原始文件
</Info>

<Info>
**向量字段**：`embedding` 字段（1536 维向量）不会在响应中返回，因为它仅用于后端检索。
</Info>

## 错误码

| HTTP 状态码 | 错误代码 | 说明 |
|-----------|---------|------|
| 401 | `UNAUTHORIZED` | 未提供有效的认证 Token |
| 403 | `FORBIDDEN` | 无权访问此知识条目（租户权限不匹配） |
| 404 | `NOT_FOUND` | 知识条目不存在或已被删除 |
| 500 | `INTERNAL_ERROR` | 服务器内部错误 |

## 常见问题

<AccordionGroup>
  <Accordion title="为什么我获取不到某个知识条目？">
    可能的原因：
    1. 知识条目 ID 错误或不存在
    2. 知识条目为 `visibility='private'`，且你不属于该租户
    3. 知识条目已被删除
  </Accordion>

  <Accordion title="如何获取知识列表？">
    使用知识列表接口 `GET /knowledge-base`，支持分页、筛选和搜索。详见知识列表接口文档。
  </Accordion>

  <Accordion title="usage_count 是什么？">
    `usage_count` 表示该知识条目被 RAG 问答系统引用的次数。每次用户提问时，如果该知识被作为上下文使用，计数会 +1。
  </Accordion>
</AccordionGroup>
